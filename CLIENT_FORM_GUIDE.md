# Руководство по форме анкеты клиента

## Страница: /dashboard/clients/new

### Структура формы

Форма разбита на 3 секции с валидацией через Zod и react-hook-form.

### Секция 1: Личные данные

- Фамилия (обязательное)
- Имя (обязательное)
- Отчество (необязательное)
- Дата рождения (обязательное, клиент должен быть старше 18 лет)
- Гражданство (обязательное): РФ / Иностранное / Апатрид

### Секция 2: Паспортные данные

- Серия паспорта (4 цифры, маска 0000)
- Номер паспорта (6 цифр, маска 000000)
- Кем выдан (обязательное)
- Дата выдачи (обязательное)
- Код подразделения (маска 000-000)
- Адрес регистрации (обязательное)
- Чекбокс "Адрес проживания совпадает" (по умолчанию включён)
  - Если выключен → появляется поле "Адрес проживания"
- СНИЛС (маска 000-000-000 00, необязательное)
- ИНН (11 цифр, необязательное)

### Секция 3: Оценка риска

- Цель займа (select, необязательное)
- Источник дохода (select, необязательное)
- Является ли ПЭП (toggle switch)
  - Если включён → появляется textarea "Опишите связь с публичной должностью"
  - Tooltip: "Публично значимые лица: чиновники, судьи, военные от определённого ранга, их ближайшие родственники"
- Группа риска (radio group, обязательное):
  - Низкий - "Стандартный клиент без признаков повышенного риска"
  - Средний - "Нерезидент, нестандартная цель займа, косвенные признаки"
  - Высокий - "ПЭП, высокорисковая страна, отказ объяснять источник средств"
  - При выборе "Высокий" → появляется textarea "Обоснование высокого риска"

### Кнопки

- "Сохранить черновик" (outline) - сохраняет без валидации, статус = 'draft'
- "Сохранить и проверить клиента" (default) - валидирует все поля, статус = 'checking'
- "Отмена" (ghost) - возврат к списку клиентов

Кнопки залипают внизу страницы (sticky bottom bar).

### Валидация

При нажатии "Сохранить и проверить":
- Все обязательные поля должны быть заполнены
- Серия: ровно 4 цифры
- Номер: ровно 6 цифр
- Дата рождения: клиент старше 18 лет
- Если is_pep = true: pep_description обязателен
- Если risk_level = 'high': risk_reason обязателен
- Если residential_address_same = false: residential_address обязателен

### Server Action

`saveClient(formData, status: 'draft' | 'checking')`:
- Получает org_id из organizations по текущему user_id
- Insert в таблицу clients
- Сохраняет snapshot в client_history
- Возвращает { id: string } для редиректа на /dashboard/clients/[id]

### Технологии

- shadcn/ui: Card, Input, Select, Textarea, Switch, RadioGroup, Button, Label, Tooltip
- react-hook-form + @hookform/resolvers/zod
- react-input-mask для масок (паспорт, СНИЛС, ИНН)
- DatePicker компонент для выбора дат
- TooltipProvider обёрнут в root layout

### Тестирование

1. Откройте /dashboard/clients/new
2. Заполните минимальные обязательные поля
3. Попробуйте "Сохранить черновик" - должно сохраниться без валидации
4. Попробуйте "Сохранить и проверить" с неполными данными - должны появиться ошибки
5. Заполните все обязательные поля
6. Нажмите "Сохранить и проверить" - должен быть редирект на страницу клиента
7. Проверьте tooltips (наведите на "?")
8. Проверьте условные поля (ПЭП, высокий риск, адрес проживания)
