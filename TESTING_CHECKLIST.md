## Тестирование функционала

### Главная страница (/)

Откройте: http://localhost:3000

- [ ] Страница загружается
- [ ] Видна кнопка "Начать работу"
- [ ] Видна кнопка "Войти"
- [ ] Нет ошибок в консоли браузера (F12)

### Регистрация (/register)

Нажмите "Начать работу" или перейдите на http://localhost:3000/register

- [ ] Форма регистрации отображается
- [ ] Поля: Email, Пароль, Подтверждение пароля
- [ ] Заполните форму:
  - Email: `test@example.com`
  - Пароль: `Test123456`
  - Подтверждение: `Test123456`
- [ ] Нажмите "Зарегистрироваться"
- [ ] Происходит редирект на /dashboard (или /login если включено подтверждение email)
- [ ] Нет ошибок в консоли

**Проверка в Supabase:**
- [ ] Authentication → Users → появился новый пользователь

### Вход (/login)

Если вас выкинуло на /login, войдите:

- [ ] Форма входа отображается
- [ ] Введите email и пароль
- [ ] Нажмите "Войти"
- [ ] Происходит редирект на /dashboard
- [ ] Нет ошибок в консоли

### Dashboard (/dashboard)

После входа вы должны быть на главной странице dashboard:

- [ ] Боковое меню отображается с пунктами:
  - Главная
  - Клиенты
  - ПВК
  - Настройки
  - Выход
- [ ] Блок "Статус Compliance" отображается
  - Профиль МФО (статус: не заполнен)
  - ПВК (статус: не обновлён)
- [ ] Блок "Статистика клиентов" отображается
  - Всего: 0
  - Одобрено: 0
  - На проверке: 0
  - Отклонено: 0
- [ ] Блок "Быстрые действия" отображается
  - Кнопка "Добавить клиента"
  - Кнопка "Обновить ПВК"

## 🐛 Если что-то не работает

1. Проверьте `.env.local` - правильные ли ключи
2. Проверьте Supabase Dashboard - активен ли проект
3. Проверьте консоль браузера - какие ошибки
4. Проверьте терминал - какие ошибки сервера
5. Перезапустите dev сервер
6. Очистите cookies браузера для localhost:3000
