## Примечания

1. Эти данные являются тестовыми и не соответствуют реальным паспортам
2. Для реальных проверок через NewDB API требуется действующий API ключ
3. Результаты проверок будут зависеть от того, что вернет API
4. Рекомендуется сначала проверить работу API на одном клиенте

## Как использовать

1. Откройте форму создания клиента: http://localhost:3000/dashboard/clients/new
2. Заполните данные одного из тестовых клиентов
3. Нажмите "Сохранить и проверить клиента"
4. Система автоматически запустит проверки через NewDB API
5. Результаты отобразятся на карточке клиента

## Проверка настроек

Убедитесь что в `.env.local` указан ключ NewDB API:
```
NEWDB_API_KEY=ваш_ключ
```

Если ключ не указан, проверки будут возвращать ошибку 422.
