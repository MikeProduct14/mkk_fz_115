## 📋 Следующие шаги

### 1. Настройте Supabase (5 минут)

Следуйте инструкции в файле `SUPABASE_SETUP_GUIDE.md`:

1. Создайте проект на https://supabase.com
2. Скопируйте URL и API ключ
3. Обновите `.env.local`
4. Выполните SQL миграцию

### 2. Перезапустите dev сервер

После обновления `.env.local`:

```bash
# Остановите текущий сервер (Ctrl+C)
npm run dev
```

### 3. Откройте приложение

Перейдите на http://localhost:3000

### 4. Протестируйте базовый функционал

1. **Главная страница** (/)
   - Должны быть кнопки "Начать работу" и "Войти"

2. **Регистрация** (/register)
   - Зарегистрируйте тестовый аккаунт
   - Email: test@example.com
   - Пароль: Test123456

3. **Вход** (/login)
   - Войдите с созданными учётными данными

4. **Dashboard** (/dashboard)
   - Должна открыться главная страница с:
     - Статусом compliance
     - Статистикой клиентов
     - Боковым меню навигации

5. **Навигация**
   - Клиенты (/dashboard/clients)
   - ПВК (/dashboard/pvk)
   - Настройки (/dashboard/settings)

## 🔍 Проверка работы

### Консоль браузера (F12)

Не должно быть ошибок типа:
- ❌ "Invalid API key"
- ❌ "Failed to fetch"
- ❌ "Supabase client error"

### Supabase Dashboard

После регистрации проверьте:
1. **Authentication** → **Users** - должен появиться новый пользователь
2. **Table Editor** → **organizations** - должна появиться запись (если реализована логика)

## 🐛 Возможные проблемы

### Проблема: "Invalid API key"
**Решение**: Проверьте `.env.local`, убедитесь что скопировали правильные ключи

### Проблема: Страница не открывается
**Решение**: 
- Проверьте, что dev сервер запущен
- Откройте http://localhost:3000 (не https)

### Проблема: Редирект не работает
**Решение**: Очистите cookies браузера для localhost:3000
