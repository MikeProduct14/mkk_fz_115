# Исправления - Краткая сводка

## Что было исправлено

### ✅ 1. DatePicker - упрощён UI
**Проблемы:**
- Не работал выбор года через dropdown
- Сложный UI

**Решение:**
- Убрана зависимость от lucide-react (использована эмодзи 📅)
- Изменён `captionLayout` с "dropdown-buttons" на "dropdown"
- Убран `asChild` из PopoverTrigger (не поддерживается Button компонентом)
- Input для ручного ввода + кнопка календаря рядом

### ✅ 2. Форма клиента - исправлен сбор данных
**Проблемы:**
- Кнопка "Сохранить и проверить клиента" не работала
- Кнопка "Сохранить черновик" выдавала ошибку
- Булевы значения не конвертировались правильно в FormData

**Решение:**
- Создана утилита `dataToFormData()` для правильной конвертации данных
- Булевы значения теперь конвертируются в строки 'true'/'false'
- Пустые значения пропускаются
- Добавлены console.error для отладки
- Исправлен onChange для DatePicker (правильный формат YYYY-MM-DD)

### ✅ 3. Обработка дат
**Решение:**
- DatePicker onChange теперь конвертирует дату в формат YYYY-MM-DD
- Используются локальные методы Date (getFullYear, getMonth, getDate)
- Убрана утилита dateToString из компонента (логика перенесена в onChange)

## Изменённые файлы

1. `components/date-picker.tsx` - упрощён UI, убрана lucide-react
2. `app/dashboard/clients/new/page.tsx` - исправлен сбор данных в FormData

## Как проверить

1. Откройте http://localhost:3001/dashboard/clients/new
2. Заполните форму:
   - Введите ФИО
   - Выберите дату рождения (через календарь или вручную)
   - Заполните паспортные данные
3. Нажмите "Сохранить черновик" - должно сохраниться без ошибок
4. Заполните все обязательные поля
5. Нажмите "Сохранить и проверить клиента" - должно сохраниться и перейти на страницу клиента

## Проверка DatePicker

1. Кликните на кнопку 📅 рядом с полем даты
2. В календаре должен быть dropdown для выбора года
3. Выберите год 1990
4. Выберите месяц и день
5. Дата должна появиться в поле ввода

Или введите дату вручную: `15031990` → автоматически станет `15.03.1990`
