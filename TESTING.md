## Исправленные проблемы

### 1. Валидация
**Проблема:** Логика refine была инвертирована для условных полей.

**Исправление:**
```typescript
// Было (неправильно):
.refine((data) => !data.residential_address_same || data.residential_address, ...)

// Стало (правильно):
.refine((data) => {
  if (data.residential_address_same === false) {
    return !!data.residential_address && data.residential_address.length > 0
  }
  return true
}, ...)
```

### 2. DatePicker
**Проблема:** 
- Дата смещалась на день назад при ручном вводе
- Нельзя было выбрать год через dropdown

**Исправление:**
- Полностью переработан для работы с UTC (избежание timezone проблем)
- Добавлены утилиты `createUTCDate()` и `parseDate()` для работы с датами в UTC
- Форматирование через UTC компоненты вместо `format()` из date-fns
- В формах добавлена утилита `dateToString()` вместо `toISOString().split('T')[0]`
- Добавлен `captionLayout="dropdown-buttons"` для выбора года
- Добавлены `fromYear={1900}` и `toYear={new Date().getFullYear()}`
- Парсинг даты при вводе в формате дд.мм.гггг
- Валидация при потере фокуса (blur)

**Подробности:** См. `DATEPICKER_FIX.md`

**Тесты:** 
- Автоматические: `tests/date-picker.spec.ts` (12 тестов)
- Ручные: `MANUAL_TESTING_DATEPICKER.md`

## Запуск тестов в CI/CD

Тесты автоматически запускаются при:
- Push в main/master
- Pull Request

Конфигурация в `playwright.config.ts`:
- Автоматический запуск dev сервера
- Retry при падении (2 раза в CI)
- HTML отчёт с результатами
- Trace при первом падении

## Добавление новых тестов

1. Создайте файл в папке `tests/`
2. Импортируйте `test` и `expect` из `@playwright/test`
3. Используйте `test.describe()` для группировки
4. Используйте `test.beforeEach()` для setup
5. Пишите тесты с понятными названиями

Пример:
```typescript
import { test, expect } from '@playwright/test'

test.describe('Новая функция', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('/dashboard/clients/new')
  })

  test('должна работать корректно', async ({ page }) => {
    // Ваш тест
  })
})
```
