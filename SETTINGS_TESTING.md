# Тестирование страницы настроек

## Быстрый старт

1. Откройте http://localhost:3000
2. Войдите в систему (или зарегистрируйтесь)
3. Перейдите в раздел "Настройки" в сайдбаре

## Тест 1: Автозаполнение по ИНН

### Тестовые ИНН для проверки:

**Активная организация (Сбербанк):**
- ИНН: `7707083893`
- Должно загрузиться: название, адрес, руководитель

**Активная организация (Яндекс):**
- ИНН: `7736207543`
- Должно загрузиться: название, адрес, руководитель

### Что проверить:

1. Введите ИНН в поле
2. Кликните вне поля (blur event)
3. Должен появиться спиннер загрузки
4. Через 1-2 секунды поля должны заполниться
5. Должен появиться toast "Данные загружены из ЕГРЮЛ"

### Ошибки:

- Неверный ИНН → "Организация не найдена"
- Ликвидированная → "Организация недействующая" (но данные всё равно загрузятся)

## Тест 2: Валидация формы

### Обязательные поля:
- ИНН (10 или 12 цифр)
- Название организации
- Тип организации (МКК или МФК)

### Опциональные поля:
- Адрес
- ФИО СДЛ
- Должность СДЛ
- Дата обновления ПВК

### Проверьте:

1. Попробуйте отправить пустую форму → должна быть ошибка
2. Введите ИНН с буквами → должна быть маска (только цифры)
3. Введите ИНН 9 цифр → "ИНН должен содержать 10 или 12 цифр"
4. Заполните только обязательные поля → должно сохраниться

## Тест 3: DatePicker

1. Кликните на поле "Дата последнего обновления ПВК"
2. Должен открыться календарь
3. Выберите дату
4. Дата должна отобразиться в формате `дд.мм.гггг`
5. Календарь должен закрыться

**Проверьте, что нет ошибки:** "button cannot be a descendant of button"

## Тест 4: Сохранение данных

1. Заполните все поля
2. Нажмите "Сохранить и продолжить"
3. Должен появиться toast "Профиль сохранён"
4. Должен быть редирект на `/dashboard`
5. На dashboard должен быть статус "Профиль МФО: Заполнен" (зелёный)

### Проверка в БД:

Откройте Supabase Dashboard → Table Editor → organizations
Должна быть запись с вашими данными.

## Тест 5: Повторное редактирование

1. Вернитесь в `/dashboard/settings`
2. Форма должна быть заполнена сохранёнными данными
3. Измените любое поле
4. Сохраните
5. Данные должны обновиться (upsert)

## Ожидаемые результаты

✅ Нет ошибок в консоли браузера
✅ Нет ошибок hydration
✅ Автозаполнение работает (если есть Dadata API key)
✅ Валидация работает корректно
✅ DatePicker открывается и закрывается
✅ Данные сохраняются в БД
✅ Редирект на dashboard после сохранения
✅ Статус профиля обновляется на dashboard

## Если что-то не работает

### Автозаполнение не работает:
- Проверьте `DADATA_API_KEY` в `.env.local`
- Проверьте консоль браузера (F12) → Network → `/api/dadata`
- Проверьте, что API route `/api/dadata/route.ts` существует

### Данные не сохраняются:
- Проверьте консоль браузера → ошибки
- Проверьте Supabase Dashboard → Authentication → Users (есть ли пользователь)
- Проверьте RLS политики в таблице `organizations`

### Ошибка "cookies() inside unstable_cache":
- Эта ошибка уже исправлена в `page-content.tsx`
- Если всё ещё появляется - перезапустите dev сервер

### Ошибка "button cannot be descendant of button":
- Эта ошибка уже исправлена в `date-picker.tsx`
- Если всё ещё появляется - очистите кэш браузера (Cmd+Shift+R)
